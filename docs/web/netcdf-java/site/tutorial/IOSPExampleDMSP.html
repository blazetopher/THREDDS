IOSP Example - DMSP

The ucar.nc2.iosp.dmsp.DSMPiosp class provides access to DMSP satellite data in the NOAA/NGDC DMSP archive format. Currently only data from the OLS instrument is supported, in particular only NOAA/NGDC DMSP OIS (OLS Integrated Smooth) data files. The OIS data is visible and thermal imagery at 2.7km resolution.

The DMSP satellites are polar orbiting satellites crossing the equator, depending on the satellite, at either dawn/dusk or noon/midnight.

More information is available at http://dmsp.ngdc.noaa.gov/.

Data Structure

One or more text header records followed by one data record per scan. Each data record contains satellite ephemeris, calibration, and quality control data followed by visible and infrared scan data.

  • Text header record (metadata)
  • data records (one per scan)
    • Time: year, day of year, seconds of day
    • Satellite ephemeris: latitude, longitude, altitude, heading
    • Sun and moon elevation, azimuth, phase
    • Gain and calibration
    • Quality control
    • Visible scan data
    • Infrared scan data

IOSP Details

The DMSP IOSP code is located in thredds/cdm/src/main/java/ucar/nc2/iosp/dmsp/DMSPiosp.java.

  • On dataset open:
    • Read header information and create dimensions and global attributes.
    • Create variables and variable attributes.
    • Make sure global and variable attributes needed for convention(s) are in place.
  • On variable read:
    • A time coordinate variable is calculated.
    • The latitude and longitude for each pixel is calculated from the per scan satellite ephemeris data.
    • Time, satellite ephemeris, and latitude/longitude data is cached.
    • Read all data for requested variable, subsetting (if requested) is done using ucar.ma2.Array subsetting.

There are a number of trade-offs made here.
