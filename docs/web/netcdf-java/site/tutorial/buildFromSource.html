Tutorial: Building From Source

Subversion

The netCDF-java source code is available from the Subversion server at:

  • [Available for Workshop] svn://zero/thredds/trunk/
  • [Not yet available] http://www.unidata.ucar.edu/svn/thredds/trunk/

This is the main trunk of the THREDDS project code repository which contains the netCDF-java, ToolsUI, and THREDDS Data Server code.

Check out code

To check out all the code from the trunk of the THREDDS subversion repositoryproject code, run the following command:

  • svn co svn://zero/thredds/trunk /your/working/directory

Workshop Update

Working directories have already been checked out on the workshop machines at "/home/workshop/TDS/code/thredds". To update that code, run these two commands:

  • cd /home/workshop/TDS/code/thredds
  • svn update

Main Ant Targets

  • compile - compiles the netCDF-Java/CDM code
  • makeJar - builds the minimal netCDF library, ncCore-*.jar, which reads netCDF-3 files; and the full netCDF library, netcdf-*.jar, which implements the full CDM.
  • compileUI - compiles the netCDF-Java/CDM and ToolsUI code
  • makeJarUI - builds the netCDF ToolsUI library, netcdfUI-*.jar, which includes the netCDF library plus the user interface classes for the netCDF ToolsUI application.

Build

To build the code needed to run the ToolsUI, run these two commands:

  • cd cdm
  • ant compileUI

Run ToolsUI

To run the netCDF ToolsUI application, run the following script:

  • target/runToolsUI
which runs the following command (with the classpath expanded):
  • java -Xmx512m -classpath <classpath> ucar.nc2.ui.ToolsUI
