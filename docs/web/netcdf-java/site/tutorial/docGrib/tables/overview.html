1.0 Overview

The Java GRIB Library can read GRIdded Binary (GRIB) files in both GRIB-1 and GRIB-2 formats. Decoding GRIB files requires a set of lookup tables for parameters, issuing centers, etc. The Java GRIB Library comes pre-configured to support commonly used tables, but regional models and modeling centers may use tables that are not in the distribution. If the parameter tables are available, there are routines in the libraries for one to read in the tables or other options available to update the GRIB jar file. One word of caution, the parameter tables have rigid formats that will be detailed in the next sections. This format was developed at NWS and it was adopted for the GRIB-1 tables but GRIB-2 only has one table that is in XML format.

If one obtains a new parameter table, Unidata would appreciate if the table and the dataset information was sent to support so this information could be shared with other members of the community.
