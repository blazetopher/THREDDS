NetCDF Java Library (Version 2)


The NetCDF-Java Library (version 2) is a Java interface to NetCDF files.  It is built on the MultiArray (version 2) package, which is a stand-alone Java package for multidimensional arrays of primitive types.

The library is freely available and the source code is released under the GNU Lesser General Public License (LGPL).

The preferred place to ask questions, discuss problems and features and get support is in the NetCDF-Java email list, which you can subscribe to from this page. Also you might want to search or browse previous netCDF-Java support questions.

Version 2.2 (alpha)

Version 2.2 requires Java 1.4 or above. The latest version of Java 1.5 is recommended.

Why do we designate this alpha? The stability and code quality is moderately good. However, until we implement the NetCDF-4 reader, we are not sure if the API might need some (probably minor) changes. We anticipate a stable NetCDF-4 format by fall of 2006.

The NetCDF-Java 2.2 (aka nj22) library implements a Common Data Model (CDM), a generalization of the NetCDF, OpenDAP and HDF5 data models. The nj22 library is also a prototype for the NetCDF-4 project, which provides a C language API for the "data access layer" of the CDM, on top of the HDF5 file format. The nj22 library is a 100% Java framework for reading netCDF and other file formats into the CDM, as well as writing to the netCDF-3 file format. The nj22 library also implements NcML, which allows you to add metadata to CDM datasets, as well as to create virtual datasets through aggregation.

Try out the library through the ToolsUI application: launch from webstart.

File types

Nj22 can read the following file types and remote access protocols:

  • NetCDF-3 (classic and large format)
  • HDF5 (most)
  • OPeNDAP client/server protocol
  • GRIB version 1 and version 2 (ongoing work on tables)
  • BUFR (limited - ongoing work)
  • GINI (GOES Ingest and NOAAPORT Interface ) image format
  • DMSP (Defense Meteorological Satellite Program)
  • NEXRAD Radar level 2 and level 3.
  • ATD's DORADE radar file format
  • ADDE client/server protocol (limited)

Features not yet available:

  • the scientific data types interfaces are still experimental, do not rely on the APIs yet.
  • writing to netCDF-3 works, but new features in the CDM, eg groups, structures, the long data type are not available yet.
  • reading and writing netCDF-4 format not done yet; writing the full CDM to netCDF will require NetCDF-4 file format.

Software libraries used:

  • Apache Commons HttpClient for HTTP access, which also use commons-logging and commons-codec.
  • Java-OPeNDAP library JDAP 1.1.7 for reading OPeNDAP, which requires gnu-regexp.
  • JDOM for parsing XML.
  • JGoodies for UI stuff
  • JPEG2000 library for decoding GRIB2 files.
  • Log4J and its companion Simple Logging Facade for Java (SLF4J) for logging
  • VisAD for reading ADDE
  • Xerces for parsing XML
