CDM File Types

The NetCDF-Java/CDM library provides a uniform API to many different scientific file formats and remote access protocols. Generally, application programmers should respect the encapsolation of these formats. When information is needed about the underlying file type, NetcdfFile.getFileType() and NetcdfFile.getFileDescription() methods can be called.

The FileType must be unique and once registered, will never change, so that application code can test against it. The Description is human-readable and may present variations when appropriate, such as adding version information, etc. The Reference URL(s) are informational, and may change as needed.

To support this functionality, getFileType() and getFileDescription() methods have been added to the IOServiceProvider interface as of version 4.0.46. You will need to add these methods to your IOServiceProvider implementations.

To register your format/IOServiceProvider, or to send corrections and additions to this table, please send email to support-netcdf-java@unidata.ucar.edu.
