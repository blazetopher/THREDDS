NetCDF Tools UI

Install and run via WebStart

  1. If you dont already have it installed, install Java 1.4 This will also install WebStart and the Java Plug-In.
  2. Launch netcdfTools with WebStart .

    3.  
  3. When first starting, you will see a window that says:

    Indicate "yes" by pressing the "Start" button.


Possible Webstart problems:

General Problem Diagnosis:

  1. Start up javaws.exe, the webstart application manager. This usually lives in a subdirectory of the jre directory, eg /home/j2sdk/jre/javaws/javaws.exe. Windows users often have a desktop icon for it. You may be able to find it using "which javaws" from the command line. Note the path, it tells you which java runtime you are using. If its pre 1.4.2, you will want to upgrade.
  2. Once its running, from the top menu, choose File > Preferences.
  3. Go to the Advanced tab and make sure Show Java Console is checked. This will cause a webstart console window to open, whenever a webstart application starts, showing error messages etc. When something goes wrong, look in that window.
  4. Go to the Java tab and check which runtime version is being used. The ToolsUI application requires 1.4.2 or above. Make sure there is a 1.4.2 runtime that is enabled. If not, you can search for one with the Find button. If you cant find one, then you need to install one, then come back here and make sure it gets enabled.

Specific Problems:

  • Problem: After clicking on "Launch," you get a message from Netscape asking if you want to save the file "netcdfTools.jnlp".
    • Diagnosis:  After installing WebStart, you must exit Netscape and start up again. If that doesn't work, log out and back on.
       
  • Problem: Upon launching, you get the error message:
            An error occurred while launching/running the application.
               Category: Launch File Error
               Unsupported JNLP version in launch file: 1.0. Only version 0.2 is supported with this version
    • Diagnosis: you have an old version of WebStart. Download the current version.
       
  • Problem:  Right after starting, the application is not responding
    • Diagnosis: There is a dialog box that has been covered by the application, that is waiting for you to dismiss it.  You have to find the dialog box and dismiss it before the app will be responsive. To uncover it, try iconifying and deiconifying other applications, until the dialog box pops to the top.
