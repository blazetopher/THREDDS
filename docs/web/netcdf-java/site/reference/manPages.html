NCDumpW

Approximate functionality to ncdump c utility, but with different arguments.

ucar.nc2.NCdumpW filepath [-cdl | -ncml] [-c | -vall] [-v varName1;varName2;..] [-v varName(0:1,:,12)]

eg: java -classpath netcdfAll-4.2.jar ucar.nc2.NCdumpW Q:/cdmUnitTest/ncml/nc/test_7.nc

where:

  • filepath : pathname of any CDM file.
  • -cdl : show CDL (strict mode)
  • -ncml : show NcML (default)
  • -c : show data for coordinate variables only
  • -vall : show data for all variables
  • -v varName1;varName2;.. : show data for these variables, using long names with ';' seperators
  • -v varName(0:1,:,12) : show data for a section of this variable only, using F90 section specificiation

FileWriter

Writes CDM classic files into netcdf-3 format. Cannot write extended data model, eg Groups, Structures, multiple unlimited dimensions, etc.

ucar.nc2.FileWriter -in fileIn -out fileOut

CompareNetcdf2

Compare 2 CDM files for semantic equivilence..

ucar.nc2.util.CompareNetcdf2 filepath1 filepath2

FeatureScan

Scans all the files in a directory to see if they are CDM files and can be identified as a particular feature type

ucar.nc2.ft.scan.FeatureScan directory
