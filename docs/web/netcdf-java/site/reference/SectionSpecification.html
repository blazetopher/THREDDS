Collections Specification

A collection specification string can be definied that creates a collection on the fly. Currently (4.1) only works for point feature collections.

Example

/data/ldm/pub/native/grid/NCEP/GFS/Alaska_191km/**/GFS_Alaska_191km_#yyyyMMdd_HHmm#\.grib1

    
    
  • rootDir= /data/ldm/pub/native/grid/NCEP/GFS/Alaska_191km
    • 
    
  • subdirs= yes
    • 
    
  • dateFormatMark= GFS_Alaska_191km_#yyyyMMdd_HHmm
    • 
    
  • onName= yes
    • 
    
  • regexp= GFS_Alaska_191km.........\.grib1

Use in CDM 4.1 library

FeatureDatasetFactoryManager.open(location, wantFeatureType) looks for collection:spec prefix for the location, and calls CompositeDatasetFactory.factory(wantFeatureType, spec) if found, which returns a FeatureDataset. only Point, Station wantFeatureType currently supported.
