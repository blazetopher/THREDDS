CDM Remote Web Service

CDM Remote is an experimental web service implemented in the CDM library (client) and TDS (server), providing remote access to CDM datasets, using ncstream as the on-the-wire protocol. It provides access at the NetcdfFile and FeatureDataset levels of the CDM API., so there are two levels of services:

  1. CdmRemote provides index subsetting on remote CDM datasets, with the same functionality that NetcdfFile provides for CDM local datasets and DODSNetcdfFile provides for remote OPeNDAP datasets.
  2. CdmRemoteFeatureDataset provides coordinate subsetting on remote CDM Feature Datasets, with similar functionality to WCS and Unidata's experimental NetCDF Subset Service (NCSS). Currently (version 4.1) it provides a remote API for Point and StationTimeSeries Feature Datasets.

The CDM Remote services and protocol are experimental and should not be used outside of the CDM stack for now.

CdmRemote Protocol (Data Access and Coordinate Systems)

ucar.nc2.stream.CdmRemote is a subclass of NetcdfFile which provides index subsetting on remote CDM datasets NetcdfDataset.openOrAcquireFile() looks for cdmremote:url prefix, and calls new CdmRemote(url) if found. The url must be an endpoint for a cdmremote service.

The client forms requests of the form url?query. The possible query parameters are:

  req=( CDL | NcML | getCapabilities | cdmHeader | data)
  var=vars

where:
  vars := varspec | varspec[,varspec]
  varspec := varname[subsetSpec]
  varname := valid variable name
  subsetSpec := '(' fortran-90 array spec ')'
  • request names are case-insensitive
  • var parameter only used for data request
  • varnames are case-sensitive and must be UTF8-escape encoded
  • literals are in bold

Example service requests are:

Request Response

http://server:8080/thredds/cdmremote/data.nc?req=CDL

dataset CDL
http://server:8080/thredds/cdmremote/data.nc?req=NcML dataset NcML
http://server:8080/thredds/cdmremote/data.nc?req=getCapabilities capabilities XML
http://server:8080/thredds/cdmremote/data.nc?req=cdmHeader header ncstream message, contains the structural metadata
http://server:8080/thredds/cdmremote/data.nc?req=data&var=Temp(0:99:10,0:127,:),lat,lon data ncstream message
  • Data request uses Section specification (same as Fortran-90 array notation) to ask for a subset in index space.
  • Variable names must be UTF8-escape encoded. In Java we use URLEncode.encode() and URLDecode().decode. This creates a application/x-www-form-urlencoded MIME encoding.
  • GetCapabilities indicates which (if any) feature types the dataset can be opened as, along with the URLs of those services. TBD.
  • The protobuf messages are defined by thredds\cdm\src\main\java\ucar\nc2\stream\ncStream.proto.

CdmRemoteFeatureDataset Protocol (Point Feature API)

ucar.nc2.stream.CdmRemoteFeatureDataset.factory() makes an HTTP request to endpoint+"?req=getCapabilities" to obtain the GetCapabilities XML document. This document describes what feature type(s) the dataset supports, which determines what are the valid requests, along with the endpoints for those feature types, which may be different than the original endpoint.

FeatureDatasetFactoryManager.open() looks for cdmremote:url prefix, and calls CdmRemoteFeatureDataset.factory(url) if found. ThreddsDataFactory.openFeatureDataset() looks for catalog InvDatasets with featureType.isPointFeatureType() and ServiceType.CdmRemote, and if found, calls CdmRemoteFeatureDataset.factory().

Possible query parameters are:

  req=( getCapabilities | data)
  time_start,time_end=time range
  north,south,east,west=bounding box
  var=vars
  stn=stns

where:
  vars := varName | varName[,varName]
  stns := stnName | stnName[,stnName]
  varName := valid variable name
  stnName := valid station name
  • request names are case-insensitive
  • variable names are case-sensitive and must be UTF8-escape encoded
  • station names are case-sensitive and must be UTF8-escape encoded
  • literals are in bold

PointDataset

Example service requests are:

Request Response Meaning
http://server:8080/thredds/cdmremote/data.nc/point?req=getCapabilities capabilities XML describe dataset
http://server:8080/thredds/cdmremote/data.nc/point?req=data pointData ncstream message get all the data
http://server:8080/thredds/cdmremote/data.nc/point?req=data&north=40.3&south=22.8&east=-80&west=-105 pointData ncstream message get data in bounding box
http://server:8080/thredds/cdmremote/data.nc/point?req=data&time_start=&time_end= pointData ncstream message get data in time range
http://server:8080/thredds/cdmremote/data.nc/point?req=data&var=Temp,lat,lon pointData ncstream message get data for listed variables
  • data requests return an unordered list of observation as StructureData
  • The protobuf messages are defined by thredds\cdm\src\main\java\ucar\nc2\ft\point\remote\pointStream.proto
