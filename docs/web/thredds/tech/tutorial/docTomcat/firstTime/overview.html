1.0 Overview

The concept is to create an user tomcat with a low permissions environment for
security reasons, therefore if a break-in should occur then the hacker cannot do 
much damage. This is the same approach that is used for an apache web server 
installations. 

There is a set of system administration steps that need to be done as root and 
a set steps for user tomcat. It is important that tomcat doesn't get more 
permissions then is necessary. Also there is a security section to secure the 
environment as tight as possible, it is suggested that one does all the security 
measures. It is impossible to close all gaps but it does help to apply the 
security tasks.
