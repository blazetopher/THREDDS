Create an Archive with the Thredds Data Repository (TDR)

The main focus of the TDS has been to serve real time data that arrives at a server by a separate mechanism such as our Unidata Internet Data Distribution (IDD) system.

The THREDDS Data Repository (TDR) is a server that complements the TDS by providing a method for clients to upload data and metadata to a TDS. The TDR provides this functionality:

  • file upload capability via HTTP (and support for gridftp)
  • a means for a client to structure the catalogs and data
  • incorporation of metadata provided by the client
  • generation of additional metadata
  • uploading a collection of data as single file

To date TDR development has been steered by two use cases. First is the creation and maintainence of a case study archive, the Next Generation Case Study (NGCS) project. Second is the needs of the LEAD cyberinfrastructure project.
