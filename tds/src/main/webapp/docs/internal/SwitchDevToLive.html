Motherlode TDS: Switching Dev to Live

last update: 24 March 2006

Running Tests

There are a number of tests that can be run before doing the switch.

Comparing Dev to Live

  • thredds.TestCompareTwoCatalogs.java compares one of these catalogs between dev and live. Edit "test/data/thredds/testCompareTwoCatalogs.config.xml" to choose which one:
    1. /thredds/catalog.xml
    2. /thredds/topcatalog.xml
    3. /thredds/idv/rt-models.1.0.xml
      • By hand look at the DQC latest stuff
    4. Test a dynamic catalog, such as /thredds/idd/nexrad/level3/NVW/YUX/20060315/catalog.xml (MODIFY the date as needed)
  • thredds/test/src thredds.TestServerSite.java

Stress Test

  • ucar.nc2.thredds.server.TestTDS : starts up several threads to access the NCEP models through opendap
  • ucar.nc2.thredds.server.TestIDVdatasets: open all datasets in http://motherlode.ucar.edu:9080/thredds/idv/rt-models.1.0.xml

The Switch

The dev Tomcat server on Motherlode uses the link /opt/tomcat9 and the live uses link /opt/tomcat. Because both dev and live probably have the same tomcat version, the switch must be done in a manner to preserve the versions. Usually dev points to /opt/jakarta-tomcat-$version-dev directory and live point to /opt/jakarta-tomcat-$version directory .

The dev Tomcat server is using ports 9080/9443/9005 and the live Tomcat server is using ports 8080/8443/8005. Switching the dev and live servers is done by shutting both servers down, changing the ports they each use and some of utility scripts in the bin directory.

  1. Become the tomcat user
    1. sudo su - tomcat
  2. Make sure everything is ready to go on dev (/opt/tomcat9/)
    1. Make a clean release
    2. Delete any cruft
  3. Shutdown servers:
    1. /opt/tomcat/bin/shutdown.sh
    2. /opt/tomcat9/bin/shutdown.sh
  4. Remove logs:
    1. rm /opt/tomcat/logs/*
    2. rm /opt/tomcat9/logs/*
    3. rm /opt/tomcat9/content/thredds/logs/*
  5. Copy things we want to save onto new server
    1. cp /opt/tomcat/content/thredds/logs/* to /opt/tomcat9/content/thredds/logs/
    2. Check if /opt/tomcat/conf/tomcat-users.xml has been changed.
    3. Check if /opt/tomcat/conf/keystore-tomcat has been changed.
  6. Become the root user
    1. su root
    2. cd /opt
  7. Move old code base:
    1. ls -dl tomcat
    2. Moved result to jakarta-tomcat-$version.old, ie
    3. mv jakarta-tomcat-5.0.28 jakarta-tomcat-5.0.28.old
  8. Create new code base, save original dev code base:
    1. ls -dl tomcat9
    2. tar results jakarta-tomcat-$version.dev, ie
    3. tar cvf devbase.tar jakarta-tomcat-5.0.28.dev
    4. Moved dev code base to jakarta-tomcat-$version, ie
    5. mv jakarta-tomcat-5.0.28.dev jakarta-tomcat-5.0.28
  9. Remove links:
    1. rm /opt/tomcat
    2. rm /opt/tomcat9
  10. Untar dev code base:
    1. tar xvf devbase.tar
    2. rm devbase.tar
  11. Create tomcat and tomcat9 links:
    1. ln -s jakarta-tomcat-5.0.28 tomcat
    2. ln -s jakarta-tomcat-5.0.28.dev tomcat9

    12. At this point tomcat points to the new server and tomcat9 point to the dev server

  12. Become the tomcat user
    1. su - tomcat
  13. Edit /opt/tomcat/conf/server.xml
    1. change 9080 to 8080 (1 place)
    2. change 9005 to 8005 (1 place)
    3. change 9443 to 8443 (2 places)
    4. Check the old server.xml to see if any changes were made to the "IP filter" Valve. looks like 
      <!-- Define the top level container in our container hierarchy -->
<Engine name="Catalina" defaultHost="localhost" debug="0">
 <Valve className="org.apache.catalina.valves.RemoteAddrValve" deny="18.83.0.150,69.25.71.12" />
  14. Modify hardcoded URLs changing 9080 to 8080
    1. perl -i.bak -pne 's#9080#8080#g' /opt/tomcat/content/thredds/dqcServlet/config/*.xml
    2. perl -i.bak -pne 's#9080#8080#g' /opt/tomcat/content/thredds/cataloggen/config/*.xml
  • Modify hardcoded bin files changing tomcat9 to tomcat
    1. perl -i.bak -pne 's#tomcat9#tomcat#g' /opt/tomcat/bin/*.sh
  • Startup servers:
    1. /opt/tomcat/bin/startup.sh
    2. /opt/tomcat9/bin/startup.sh
  • Remove Temporary Files once server is running correctly:
    1. rm -r jakarta-tomcat-5.0.28.old
