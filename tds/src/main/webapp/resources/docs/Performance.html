Performance

last update: July 09, 2005

Java Virtual Machine

  • You should give the JVM as much memory as possible.
  • You should run with the Java HotSpot Server VM.

To do so, set the Java runtime parameters with the environmental variable JAVA_OPTS. For example, add the following to ${tomcat_home}/bin/startup.sh : 

JAVA_OPTS="-server -Xmx512m -Xms128m"
export JAVA_OPTS

starts Tomcat with the Server JVM, and gives it 128 Mbytes of memory to start with, and allows a maximum of 512 Mbytes. These are suggested numbers, use more if you can. You will need to restart Tomcat for this to take effect.

Tomcat

Version

Tomcat 5.0 should have better performance than Tomcat 4.X. We recommend 5.0.28 as the best stable version available as of the date of this writing.

Tomcat 5.5 may have better performance than 5.0, but we have not tested. We hope to do some performance tests soon and give more specific recommendations. If you are in the position to do so, please let us know the results!

Automatic Startup

In a production environment, Tomcat should be automatically restarted when the machine starts. How to do this depends on what OS you are running. This FAQ has a bit of info.

Thredds Data Server

Memory Use

The OPeNDAP-Java layer of the server currently has to read the entire data request into memory before sending it to the client (we hope to get a streaming I/O solution working eventually). Generally clients only request subsets of large files, but if you need to support large data requests, make sure that the -Xmx512m parameter above is set accordingly.

File Handles

The OPeNDAP-NetCDF server will keep up to 100 NetCDF files open at a time. This is set in {tomcat_home}/webapps/thredds/WEB-INF/web.xml file, in the parameter called maxNetcdfFilesCached. You can change it, but beware that it gets overwritten when a new thredds.war is installed.
