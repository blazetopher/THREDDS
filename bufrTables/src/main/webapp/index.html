BUFR Tables Home

Background

Unidata is developing a 100% Java library for reading BUFR records (editions 2,3,4), as part of the Netcdf-Java / CDM (Common Data Model) library. For BUFR to be used across institutions or to archive data, we need canonical, standardized, machine-readable BUFR tables, shared between producer and client software. A missing table entry makes an entire BUFR message unreadable. An incorrect entry may make the BUFR record unreadable, or worse, silently give incorrect data values.

We are publishing our BUFR tables in machine-readable format along with a validation service that allows users to upload BUFR records and test them against our reader. In this way we hope that centers/users will help verify that our tables and software are correct, validate their BUFR formats, and contribute their own local tables as needed. Ideally a canonical repository of machine-readable BUFR tables will eventually be hosted by the WMO.

Standard WMO Tables

Specific areas of the WMO tables B and D are reserved for local data descriptors, namely categories 48-63 inclusive, and entries 192-255 inclusive (cf p L3-49 of the BUFR Guide). All other entries in tables B and D are reserved for globally shared descriptors, which we call standard WMO data descriptors. The standard WMO tables are available from the WMO in Microsoft Word and PDF format. Extracting tables from these formats is an error-prone and time-consuming process, that is to say, neither Word nor PDF is machine readable in this context.

We have extracted the standard tables, and validated them against an independent effort by Gil Ross from the British Met Office.

  • We assume that there is no local override of the standard WMO data descriptors
  • We assume that there is no dependency on the edition number of the BUFR record
  • We assume that all previous versions of the standard tables are backwards compatible with the latest version (currently at version 13)
  • We expect that future versions, starting with version 14, will not be compatible, and so versioned archives will need to be maintained.
